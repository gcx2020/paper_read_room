pi0.7: a Steerable Generalist Robotic Foundation Model with Emergent Capabilities

这篇论文的核心不是提出一个全新的机器人网络结构，而是把“更丰富、更可控的 prompt”作为训练配方：用语言子任务、视觉子目标、episode 质量/速度/错误元数据和控制模式，把大规模异质机器人数据、失败数据、自治 rollout、人体视频和网页多模态数据放进同一个 VLA 训练框架。

一句话贡献pi0.7 通过多模态上下文条件化，让一个约 5B 参数的通用机器人模型在无需任务级微调时接近专门 RL/SFT policy，并表现出跨任务、跨场景、跨机器人组合泛化。
研究问题当机器人数据来源混杂、质量不一、策略不同甚至包含失败时，普通 imitation learning 会平均掉多模态行为；如何让模型既吃下这些数据，又在测试时被精确 steer 到高质量策略。

背景动机

机器人 foundation model 的目标和大语言模型类似：把多来源数据压进一个通用模型，让模型在新场景中组合已有能力。问题在于机器人数据比文本更难“混”：同一个任务可能有快慢不同的执行、成功和失败、不同机器人 morphology、不同控制空间，以及从人类视频或网页多模态数据中来的非机器人信号。

论文指出，天真合并这些数据会让 policy 平均不同策略与质量层级，结果往往是低质量的折中动作。pi0.7 的核心假设是：不是先把数据清洗成单一高质量分布，而是把“这段数据是如何执行的”也放进 prompt。这样训练时保留异质性，测试时再用元数据和子目标把模型 steer 到高速、高质量、无错误的模式。

核心瓶颈

输入与输出输入是多视角相机历史、机器人 proprioception、任务/子任务语言、可选视觉子目标、episode 元数据与控制模式；输出是 50 步 continuous action chunk，实际只执行其中一部分后重新推理。
baseline 失败模式只用短文本任务 prompt 的 VLA 很难区分数据质量、执行速度、策略选择和失败片段；在复杂语言引用、数据集偏置、跨机器人 morphology、长程新任务上容易失效。
问题本质：pi0.7 把“多样性带来的歧义”转化为“可条件化的上下文变量”。如果条件变量足够贴近行为差异，模型就能从更大、更杂的数据中学习，而不是被低质量数据拖垮。

方法地图

概念示意图，非论文原图。它只表达已从正文确认的数据流：多视角历史观测与 proprioception 进入 MEM/Gemma3 VLM backbone；prompt 额外包含子任务、元数据、控制模式和可选视觉子目标；860M action expert 用 flow matching 生成动作 chunk。

训练时，数据来自高质量示范、低质量示范、失败 episode、自治评估 rollout、开源机器人数据、egocentric human video、网页 VQA/图像属性/文本任务等。每条轨迹不仅监督动作，还带有描述执行方式的上下文。推理时，系统始终给出控制模式和元数据；子任务可由人类 coaching 或高层语言 policy 产生；视觉子目标由轻量 world model 异步生成并定期刷新。

核心方法精读

1. Diverse Context：让异质数据可学习

最关键的设计是把上下文从 “Task: clean the kitchen” 扩成多字段 prompt。子任务 instruction 指出当前语义阶段；subgoal images 描述短期未来视觉状态；episode metadata 标注速度、质量和错误；control mode 告诉模型当前动作空间是 joint-level 还是 end-effector。训练时随机 dropout 每个组件，使模型测试时能接受不同组合。

Prompt 组件解决的问题训练/推理机制风险
Subtask instruction长程任务缺少阶段性语义约束来自人工标注、human coaching 或高层 language policy；可随时间变化高层指令错误会把低层 VLA 带偏
Subgoal images语言难表达抓取角度、物体布局、腕部视角等细节训练用真实未来帧与 world model 生成图；推理用 BAGEL 系 world model 生成生成图质量差时可能造成错误空间 grounding
Episode metadata失败/慢速/低质量轨迹和高质量轨迹混在一起速度离散到 500-step bin，质量 1-5 分，错误 segment 人工粗标；推理设高质量、无错误元数据标注噪声会影响行为模式分离
Control mode不同机器人和任务可能需要不同控制空间训练同时包含 joint 和 end-effector action，用文本标识控制模式；该字段不做 dropout控制器/IK 误差仍会进入实际执行

2. Flow-Based VLA：动作生成仍是连续 action expert

论文沿用 flow/diffusion 风格的 VLA：VLM backbone 处理视觉、语言与上下文，轻量 action expert 读取 backbone activation 并预测连续动作 chunk。pi0.7 的 action expert 是 860M 参数 transformer，固定处理 50 个 action tokens，对应 50 步动作；token 之间双向注意力，并可 attend 到 VLM activation。

论文 Equation (1)，HTML 中公式符号缺失处按上下文整理
log p(a | o, c) approx log-likelihood lower bound from flow matching
含义：给定 observation o 与 context c，action expert 学习动作 chunk a 的条件分布；flow matching 优化的是近似下界，不是闭式 log-likelihood。

这解释了为什么论文反复强调“steerable”而不是“planner”：动作空间仍由一个连续生成模型覆盖，prompt 的作用是改变条件分布，使模型在多种可行行为中选择符合速度、质量、子目标和控制模式的那一支。

3. MEM + Block-Causal Mask：让历史、目标和文本各用合适注意力

模型最多输入四路相机：front、两个 wrist、可选 rear；每路最多六帧历史，图像 resize 到 448x448。MEM 风格 vision encoder 对时空历史做压缩，使任意数量历史帧输出固定 token 数。观测 token 与 subgoal token 内部使用双向注意力，goal-image token 还能 attend observation；后续文本 token 使用 causal attention。proprioceptive state 通过线性投影成为 backbone token，不再离散成文本 token。

为什么有效：历史帧解决记忆类任务，subgoal token 解决短期目标 grounding，block-causal mask 避免把视觉/目标 token 和后续文本混成不合适的因果结构。

4. World Model 生成视觉子目标

subgoal images 不是静态人工图片，而是可由 world model 在推理时生成。该 world model 初始化自 BAGEL，一个 14B mixture-of-transformers 图像理解、编辑和生成模型；训练目标是根据当前多视角输入和子任务 instruction 生成短期未来图像。论文强调，web-scale 预训练带来的语义和物理概念能通过生成的视觉子目标转移给 VLA。

论文 Section V-B，world model objective 的理解式
L_wm = L_flow(g_future ; observation, subtask, metadata)
g_future 表示未来 subgoal image；论文将其作为标准 flow matching loss，用高质量 subtask label 的 trajectory segment 训练。

5. Runtime Prompting：异步高层策略、视觉目标与动作 chunk

测试时默认 metadata 设置为任务相关 speed percentile、quality=5、mistake=false。子任务来自人类 coaching 或高层 language policy；视觉子目标在语义 intent 改变或计时到达时刷新；VLA 用最新可用上下文异步推理。所有实验使用 5 个 denoising steps 生成 50-step action chunk，并执行其中若干步后重新推理。

实验设置

实验不是一个单一 benchmark，而是一组真实机器人评估：高灵巧长程任务、需要 memory 的任务、开放环境指令跟随、复杂引用指令、逆数据集偏置任务、跨 embodiment transfer、组合泛化新任务，以及数据质量/多样性消融。机器人包括 bimanual mobile manipulators、静态 BiPi bimanual、bimanual UR5e 和单臂 6 DoF 系统。

如何读指标：seen dexterous tasks 常看 success rate、normalized throughput、task progress；instruction following 看正确执行总指令比例；跨 embodiment 同时看 task progress 与 success。不同任务难度、时长和机器人平台不同，不能把所有百分比当作同一 benchmark 上的统一分数比较。

主要结果

结果 1：一个通用模型能否接近专门 policy？

Figure 6 的结论是：同一个 pi0.7 在 espresso、box building、laundry folding、peanut butter sandwich、shirt inside-out、drive through door、cut zucchini、peel fruits/vegetables、take out trash 等任务上，无需任务级 post-training 就接近 RL/SFT specialists；在 diverse laundry folding 和 box building 的 throughput 上甚至更高。Figure 7 进一步说明，metadata 和 autonomous evaluation data 都不是装饰，去掉任何一项都会掉性能，吞吐差距尤其明显。

结果 2：语言跟随是否真的更强？

Figure 9 在 4 个 unseen kitchens 和 2 个 unseen bedrooms 中测试 14 个 instruction following scenarios，每次包含 3-6 步开放指令；pi0.7 明显超过 pi0 与 pi0.5。Figure 10 把指令分成普通重排和复杂引用，后者包括“拿起我会用来喝汤的物体”之类语义引用；pi0.7 在复杂指令上优势更明显，使用 generated subgoal images 的 GC 版本进一步提升。Figure 11 的逆偏置任务说明模型不只是复现数据集常规动作，而能在 prompt 要求和数据偏置相反时仍跟随语言。

结果 3：跨机器人 transfer 与组合泛化

Figure 12/13 关注 cross-embodiment。更简单的重排任务中多个模型都能 transfer；当 morphology gap 变大，尤其是小型静态 bimanual 到 UR5e 或单臂 UR5e 的迁移，pi0.7 明显强于 prior models。论文还报告 UR5e shirt folding 中，人类 expert teleoperators 零样本 task progress 为 90.9%、success 80.6%，pi0.7 为 85.6%、success 80%，说明其零样本跨 embodiment 表现接近专家首次尝试。

组合泛化部分，Figure 15/16 展示 air fryer、toaster、bagel 等未收集 action-level data 的长程厨房任务。直接 zero-shot prompt 不一定完成，但 language coaching 能显著提升；用 coaching data 训练高层 language policy 后，可在无额外 teleoperation 的情况下接近 coaching episode 的表现。

消融分析

Q1：元数据真的能让混合质量数据变成资产吗？

Figure 18 左侧把 “Laundry (T-Shirts and Shorts)” 数据按质量和速度切成 top 30%、top 50%、top 80%、all 四档。无 metadata 的模型随着更多低质量数据进入可能变差；带 metadata 的 pi0.7 则随数据规模继续提升。这是全文最关键的因果证据之一：prompt 不是为了推理时花哨控制，而是在训练时把数据分布的多模态结构显式化。

Q2：任务多样性本身是否重要？

Figure 18 右侧比较移除“最高任务多样性 20%”与随机移除 20%。如果只是数据量，两个 ablation 应该接近；论文报告 full model 和 random 20% removal 明显优于移除最高多样性 20%，说明任务多样性对 unseen short-horizon generalization 有独立贡献。

Q3：视觉子目标在什么地方最有价值？

Figure 10/11/12/15 的共同模式是：GC 不一定在所有简单任务上必要，但在复杂引用、逆数据集偏置、跨 embodiment dexterous transfer 和长程 coaching 任务中尤其重要。直觉是，语言给出语义意图，world model 把语义意图投射成“下一小段视觉上应该是什么样”，低层 VLA 再把它转成动作。

可视化与失败模式

论文的可视化主要通过机器人照片、执行序列和实验柱状图呈现。Figure 13 最有解释力：在 source robot 上，人类示范用一只手撑开袋子或以倾斜末端执行器固定布料；在 UR5e target robot 上，pi0.7 采用更适合长臂/不同 gripper 的单臂 pick-and-place 或垂直 grasp。这个现象支持“跨 embodiment 不是复制轨迹，而是重组策略”的主张。

失败模式则集中在 unseen 长程任务：论文明确说，直接要求机器人 zero-shot 完成 air fryer 这类任务时会有合理尝试但不能完整完成；需要 step-by-step language coaching 或高层 policy 才能稳定。讨论章节也承认，seen tasks 常超过 90% success，而 unseen tasks 或 unseen task-robot 组合常在 60-80% 区间。

优点、不足与隐含假设

优点方法抓住了机器人数据的真实问题：数据大而杂、质量不一、任务多样但标注不完全。用 prompt 分解行为差异比强行过滤数据更适合扩展。
不足论文是大规模真实系统论文，很多数据细节、训练算力、标注流程和精确数值难以复现；结果多为自建任务和自家 robot fleet，外部可比性有限。

隐含假设包括：metadata 能充分刻画质量/策略差异；world model 生成的 subgoal 与真实可执行未来一致；高层 policy 或 human coaching 不会持续给错子任务；训练集中虽无目标任务 action-level data，但可能存在可重组的近邻技能；真实部署中 asynchronous world model、RTC、PD controller 和不同机器人动力学不会产生不可控误差。

后续研究与工程启发

研究上，pi0.7 给出的方向是“把数据异质性显式 prompt 化”。后续可以更系统地研究 metadata 的粒度、自动标注质量、failure segment 的利用方式，以及 world model subgoal 的可信度校准。另一个开放问题是如何定义 truly unseen：大规模数据中技能可能以别名或子行为形式出现，组合泛化和检索式重组之间的边界很难划清。

工程上，复现优先级应该是：先做可控 metadata schema，再做 prompt component dropout，然后加入 history encoder 与 action expert；视觉子目标很强但成本高，论文附录给出的 worst-case VLA inference 为 127ms，world model 25 denoising steps 约 1.25s 且用 4xH100 tensor parallel，因此更适合异步规划提示，而不是每个控制周期同步依赖。

总结

pi0.7 最核心的贡献，是证明大规模机器人 VLA 的泛化瓶颈不只是模型大小，而是训练上下文是否足以解释异质数据。最有说服力的证据来自三组结果：Figure 6/7 的通用模型接近 specialist 且 metadata/eval data 必要；Figure 12/22 的跨 embodiment dexterous transfer 接近专家首次尝试；Figure 18 的元数据让更多混合质量数据继续带来收益。最大局限是复现门槛高、外部 benchmark 少，且真正“未见任务”的界定仍然模糊。
