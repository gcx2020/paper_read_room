arXiv:2602.10098v2 / Vision-Language-Action Pretraining

VLA-JEPA: Enhancing Vision-Language-Action Model with Latent World Model

这篇论文的核心不是再发明一个动作头，而是重新定义“从视频学 latent action”这件事：未来帧只进入 frozen target encoder 生成监督目标，学生路径只能看当前观测和指令，在 latent world model 中预测未来状态，从而避免把 future image 直接压进 latent action 的信息泄漏。

一句话贡献提出 leakage-free state prediction 的 VLA-JEPA，把 JEPA 式 latent-space prediction 用作 VLA 预训练，再用 flow matching action head 做机器人动作生成。
研究问题现有 latent-action VLA 往往把帧差、背景运动、视角变化和 future-frame shortcut 学成“动作”，导致表示可预测但不一定可控。
主要结果在 LIBERO 平均 97.2、LIBERO-Plus 平均 79.5，并在 SimplerEnv Google Robot 平均 65.2；真实 Franka 实验中 ID 和 layout OOD 最优。

背景动机

VLA 模型想利用互联网规模视频来减少机器人动作标注依赖。问题在于：视频里没有真实机器人控制量，很多方法只能先学一个 latent action，再把 latent action 对齐到机器人动作。论文认为这一路线的关键风险是“学到的 latent action 不像动作”：它可能更像压缩后的帧差、光照变化、相机运动、桌布纹理或未来图像本身。

作者把现有 latent-action pretraining 的失败模式归纳为四类：像素级目标偏向 appearance；真实人类视频放大 camera motion 和背景噪声；把当前帧和未来帧同时送入学习器会产生 information leakage；多阶段训练让表示、latent action 和 policy learning 之间出现阶段错配。

核心瓶颈

输入与输出预训练输入是当前多视角观测、语言指令和 learnable latent tokens；未来视频帧只通过 V-JEPA2 target encoder 生成 latent target。机器人微调时额外输出连续动作轨迹。
baseline 失败模式LAPA/UniVLA 等 latent-action 方法可能过度关注帧间像素变化或无关语义背景；这种表示在训练 loss 上有用，但不一定代表可控状态转移。
问题本质：论文要把 latent action 从“解释未来图像长什么样”改成“解释状态如何朝任务相关方向变化”。VLA-JEPA 的建模假设是：如果 future 只作为 latent target，而不作为学生路径输入，模型就更难走 future-frame shortcut。

方法地图

概念示意图，非论文原图。它只表达论文 Figure 2 和方法章节确认的数据流：human video 只用 alignment loss 学 latent world dynamics；robot data 同时使用 latent alignment 和 flow action loss。论文原图未能本地抽取，因此这里不称为论文 Figure。

训练路径可以理解成两段。第一段在 action-free human videos 上学 latent state transition：V-JEPA2 encoder 产生未来状态 latent target，VLM 只看初始观测和语言，通过 latent tokens 生成 transition representation，再由 world model 预测未来 latent states。第二段在 robot demonstrations 上保留这个 world-model alignment，同时加入 conditional flow matching action head，让 latent transition representation 成为连续动作生成的条件。

核心方法精读

1. World State Encoder：把未来监督从像素移到 latent state

论文先用 frozen V-JEPA2 encoder 把每个视角的帧编码为状态表示，再把多视角表示拼接成统一 world state。这里的关键不是 encoder 名字，而是监督对象：模型不重建 future pixels，而是对齐 future latent state。这样可以弱化光照、纹理、相机抖动等高方差低控制因素。

论文 Equation (1)
s_{t_i} = ||_v F(I_{v,t_i})
F 是单视角 V-JEPA2 encoder，|| 表示按视角拼接。s_{t_i} 是第 i 个时间点的统一 world-state representation。

2. Leakage-Free Latent Action：未来只做 target，不做 input

VLM backbone 采用 Qwen3-VL，并加入 learnable latent tokens。对每个时间步，VLM 只基于初始多视角观测和语言指令，把 latent token 映射为 transition representation。未来帧不进入 VLM 输入；它只通过 frozen target encoder 形成监督目标。这一点直接针对现有 latent-action pipeline 中“latent action 变成 future image 压缩码”的 shortcut。

论文 Equation (2) 与 (3)
z_{t_i}=p^{VLM}_{theta}(<latent_i> | {I_{j,t_0}}_{j=0}^{v}, ell)
hat{s}_{t_{1:i+1}}=p^{WM}_{theta}(s_{t_{0:i}}, z_{t_{0:i}})
z_{t_i} 是 latent action token 对应的状态转移表示；world model 用历史 latent states 与这些 z 预测下一段 future states。

时间注意力也服务于这个目标：同一时间步内 latent/action/world-state tokens 双向注意；跨时间步严格 causal，只允许看当前和过去。这让 world model 保留“状态随时间推进”的结构，而不是任意读未来。

3. JEPA Alignment Loss：把预测目标收敛到语义 latent 空间

作者从 JEPA 角度解释 objective：frozen target encoder 提供确定性的 latent target，online predictor/world model 输出预测状态。论文先写成语义空间 predictive log-likelihood 的 ELBO/KL 形式；由于 target encoder 是 deterministic embedding，实际训练退化为 latent-space reconstruction/alignment loss。

论文 Equation (4) 与 (5)
-D_KL[F(.) || p^{WM}_{theta}]
L_WM = sum_{k=1}^{T} E_{s_{t_k}~F(.)}(hat{s}_{t_k} - s_{t_k})
论文公式未显式写范数；工程上应理解为预测 latent state 与 target latent state 的对齐损失。它约束 latent token 必须解释未来状态转移。

4. Robot Action Head：latent world dynamics 条件化 flow matching

在机器人数据上，模型追加 embodied action tokens，VLM 基于当前观测、语言和 latent tokens 生成全局动作条件表示 \(z_a\)。动作头采用 DiT-B 风格的 conditional flow matching，把高斯噪声沿时间插值积分到连续动作轨迹。最终训练目标是 action flow loss 加上 world-model alignment loss。

论文 Equation (6)-(9)
z_a=p^{VLM}_{theta}(<action> | {I_{i,t_0}}_{i=0}^{v}, ell, <latent_i>)
a_t=(1-t)epsilon + t a_{0:H}, t ~ U(0,1)
L_FM=E[ || v_theta(a_t,t | z_a) - (a_{0:H}-epsilon) ||_2^2 ]
L = L_FM + beta L_WM
z_a 把 latent transition 表示转成动作生成条件；L_WM 继续约束状态转移语义，L_FM 负责把条件映射到连续 end-effector action。

实验设置

实验覆盖三类模拟评估和一类真实机器人评估。LIBERO 是 in-distribution simulation；SimplerEnv 是 real-to-sim gap OOD，含 Google Robot 和 WidowX；LIBERO-Plus 通过 camera、robot、language、light、background、noise、layout 七类扰动测 VLA 鲁棒性；真实实验使用 Franka Research 3 + Robotiq 2F-85，100 条三任务示范做 post-training，并评估 ID、task OOD、object-layout OOD。

如何读指标：所有核心表格都是 success rate，越高越好。LIBERO 每个 suite 50 episodes per task、500 episodes per suite；LIBERO-Plus 的七列不能简单等价，因为每列扰动测试的失败来源不同；SimplerEnv 的 Google Robot 与 WidowX 是不同 embodiment，不应跨列直接比较绝对难度。

消融分析

Q1：human video 到底带来了什么？

Table 1-3 的 w/o human videos 结果说明，人类视频不是直接替代机器人控制数据。LIBERO 平均从 96.1 提到 97.2，LIBERO-Plus 从 62.9 提到 79.5，增强主要体现在扰动鲁棒性；但 SimplerEnv 上 w/o human videos 在 Google Robot 平均反而更高。作者据此判断：人类视频主要强化已有技能的鲁棒性和稳定性，例如 repeated grasping，而不是直接教授机器人动力学。

Q2：统一预训练是否比多阶段 latent-action pipeline 更合理？

Figure 6 的注意力可视化用于回答这个问题。LAPA 的 latent actions 关注过密视觉信息，包含操作无关桌面细节；UniVLA 通过文本语义缓解一部分问题，但会过度关注静止背景物；VLA-JEPA 更聚焦机械臂、手和被操作物体。这个证据支持论文主张：leakage-free latent state prediction 让 latent tokens 更接近 action-relevant transition，而不是 image compression。

概念示意图，非论文原图。真实 Figure 6 是 latent action tokens 到 image tokens 的注意力热图；本文只用它解释作者的文字结论，不复刻或冒充原图。

Q3：future video horizon 应该多长？

Table 4 显示 T=8 平均最好。T=4 信息不足，LIBERO-10 只有 89.0；T=16 引入冗余，Spatial 下降到 92.8。作者的解释是 latent action token 数等于 frame 数减一，未来 horizon 应接近 action horizon：太短不能覆盖动作动态，太长则混入无关变化。

可视化与失败模式

论文可视化主要有三类：方法架构图、实验设置图、注意力热图和真实机器人结果图。由于本地未能抽取原图，本文不嵌入这些论文 Figure。就内容而言，Figure 6 最关键，因为它直接说明 VLA-JEPA 的 latent action 更关注操作相关区域；Figure 4 的真实实验则揭示了一个重要 trade-off：VLA-JEPA 的轨迹更稳定且少触发安全边界，但细粒度语言理解弱于部分 baseline，可能抓错指令对象。

真实实验还报告了 repeated grasping：VLA-JEPA 在一次抓取失败后会重新张开夹爪再尝试，而 π0 和 π0.5 没有观察到这个技能。作者把它归因于人类视频中存在大量重复抓取知识；这也侧面说明人类视频对“策略鲁棒性和恢复行为”的贡献大于对精确机器人动力学的贡献。

优点、不足与隐含假设

优点目标函数对准了 latent-action VLA 的根本风险：信息泄漏和像素捷径。方法简化成 JEPA pretraining + action-head finetuning，比多阶段 codebook/latent alignment pipeline 更清楚。
不足论文未充分量化推理延迟、world model 预测质量与动作安全的因果关系；真实实验只有 100 条示范和三类任务，规模仍小。

隐含假设包括：V-JEPA2 latent target 足够 action-relevant；human videos 的状态转移可迁移到机器人控制；future horizon 与 action horizon 的关系在不同帧率和任务中仍成立；LIBERO/LIBERO-Plus/SimplerEnv 的 success rate 能代表真实部署鲁棒性。尤其需要注意，SimplerEnv 中 w/o human videos 的结果并不弱，说明人类视频不是无条件增益。

后续研究与工程启发

研究上，VLA-JEPA 值得沿三个方向继续验证：更强 target encoder 是否能提供更 action-centric 的 latent state；不同 future horizon、latent token 数和 action horizon 的耦合关系；在更复杂语言组合和动态交互场景中，leakage-free latent world model 是否仍比 subgoal image 或 explicit planner 更稳。

工程复现优先检查四件事：未来帧是否完全隔离在 target encoder 路径；world model 的 causal mask 是否正确；latent-action tokens 与 action tokens 的重复数量是否匹配论文配置；flow action head 的 action normalization、denoising timesteps 和 horizon 是否与数据控制频率一致。上线评估还应补充安全边界、错误恢复、长程任务失败原因和 OOD 语言指令测试。

总结

VLA-JEPA 的核心贡献，是把 latent-action pretraining 从“预测/压缩未来图像变化”改成“无泄漏地预测未来 latent world state”。最有说服力的证据是 LIBERO-Plus 平均 79.5 的鲁棒性结果、Figure 6 的注意力分析和 T=8 horizon 消融。最大局限是人类视频收益并非在所有 benchmark 都稳定，真实机器人实验规模较小，且 latent target 是否真正等价于可控状态转移仍需更强验证。
