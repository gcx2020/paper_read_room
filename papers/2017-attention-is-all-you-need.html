Attention Is All You Need

Transformer 的核心主张不是“在 RNN 上再加一个 attention”，而是把序列转导模型的主干完全换成 attention：编码器和解码器都不再使用循环或卷积，只依靠多头自注意力、位置编码、逐位置前馈网络以及残差归一化完成机器翻译。

一句话贡献证明全注意力架构可以在机器翻译中同时提高训练并行度、缩短长依赖路径，并达到当时最强的翻译质量。
研究问题如何保留 encoder-decoder 和自回归生成框架，同时摆脱 RNN 样本内部按时间步串行计算的瓶颈。

背景动机

论文发表前，神经机器翻译主流是 encoder-decoder：编码器把源句映射为连续表示，解码器按 token 自回归生成目标句。强模型通常依赖 LSTM/GRU 或 CNN，再用 attention 读取源句不同位置。Attention 在这些模型里更像一个辅助对齐模块，而不是整个序列模型的骨架。

真正的痛点来自计算图。RNN 的第 t 个隐藏状态依赖 t-1，训练时同一个样本内部很难并行；CNN 可以并行，但远距离 token 需要经过多层卷积传播。Transformer 的动机是把跨位置通信改成一次全局内容寻址：任意两个位置在同一层即可相互访问，顺序信息则通过显式位置编码注入。

读法提示：本文的关键不是单个 attention 公式，而是把“时间递推”替换为“全局矩阵计算”。这解释了为什么它既是建模论文，也是一次硬件友好计算图的重构。

核心瓶颈

输入与输出输入是源 token 序列，输出是目标 token 序列。训练时 decoder 输入为右移后的目标序列；推理时第 i 个目标 token 只能依赖已经生成的前缀和源句表示。
baseline 失败模式RNN 路径长度随序列长度线性增长且难并行；CNN 路径长度随层数增长；未缩放 dot-product attention 在高维下 softmax 容易饱和。

论文的建模假设可以概括为三句话：顺序不必由 recurrence 承载；位置可以作为输入特征注入；跨 token 的信息混合由 self-attention 完成，token 内部非线性变换由逐位置 FFN 完成。这一假设后来成为大语言模型、视觉 Transformer 和多模态模型的共同骨架。

方法地图

论文原图 Figure 1：Transformer 架构。阅读时从底部开始：输入 token 与右移目标 token 先经过 embedding 并加上 positional encoding；左侧 encoder 堆叠 N 层，每层是 self-attention + FFN；右侧 decoder 堆叠 N 层，先做 masked self-attention，再通过 encoder-decoder attention 读取源句表示。该图支持论文主张：序列转导的主干可以由 attention 和逐位置 FFN 构成，而不需要 recurrent state。

完整 pipeline 是：源句进入 encoder 后产生一组上下文化 memory；目标端训练时输入右移后的 ground-truth 前缀，推理时输入已生成前缀；decoder 的第一层 attention 用 mask 屏蔽未来位置，第二层 attention 以 decoder 状态为 query、encoder 输出为 key/value 读取源句；最后用线性层和 softmax 预测下一个 token。

核心方法精读

1. Scaled Dot-Product Attention：把对齐变成矩阵运算

Attention 接收一组 query、key、value。每个 query 与所有 key 做点积得到权重，再对 value 加权求和。Transformer 把所有位置的 query/key/value 组织成矩阵 Q,K,V，一次矩阵乘法就能计算所有位置之间的兼容性。

论文 Equation (1)
\[\mathrm{Attention}(Q,K,V)=\mathrm{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V\]
LaTeX: \mathrm{Attention}(Q,K,V)=\mathrm{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V

缩放项 1 / √d_k 是一个很实际的稳定性设计：当维度变大时，点积方差变大，softmax 更容易落入梯度很小的饱和区域。工程实现上，这个模块主要是 GEMM、mask、softmax、GEMM，因此比逐步 recurrent computation 更适合 GPU/TPU 批量并行。

2. Multi-Head Attention：让不同关系在不同子空间里被读取

单个 attention head 只能产生一次加权平均，容易把词法、句法、长距离依赖等关系混在一起。多头注意力先把 Q,K,V 投影到多个低维子空间，在每个子空间独立做 attention，再拼接并投影回模型维度。

论文 Section 3.2.2
\[\mathrm{MultiHead}(Q,K,V)=\mathrm{Concat}(\mathrm{head}_1,\ldots,\mathrm{head}_h)W^O\]
\[\mathrm{head}_i=\mathrm{Attention}(QW_i^Q,KW_i^K,VW_i^V)\]
LaTeX: \mathrm{MultiHead}(Q,K,V)=\mathrm{Concat}(\mathrm{head}_1,\ldots,\mathrm{head}_h)W^O

base 模型使用 h=8，d_model=512，每个 head 的 d_k=d_v=64。由于每个 head 降维，多头的总计算量接近单个全维 attention。Table 3 的消融显示 1 head 只有 24.9 BLEU，而 base 的 8 heads 为 25.8 BLEU；这说明多头不是装饰，而是在固定计算量下提供了更丰富的关系子空间。

3. 三类 attention：同一个算子承担三种角色

Transformer 里有三种 attention。encoder self-attention 让源句每个位置读取全部源 token；decoder masked self-attention 让目标位置只读取自己和历史目标 token；encoder-decoder attention 则用 decoder 状态作 query，读取 encoder 输出的源句 memory。mask 不是正则化，而是训练时保持自回归因果性的硬约束。

工程重点：如果 mask 方向错了，训练损失可能仍然下降，但模型会偷看未来 token，推理时立刻暴露分布不一致问题。这是复现 Transformer 最常见的致命错误之一。

4. Position-wise FFN：跨位置通信之后做通道变换

Attention 负责 token 之间的信息流动，FFN 负责每个位置内部的非线性通道混合。它对每个位置独立应用，并在所有位置共享参数，因此等价于 kernel size 为 1 的两层卷积。

论文 Equation (2)
\[\mathrm{FFN}(x)=\max(0,xW_1+b_1)W_2+b_2\]
LaTeX: \mathrm{FFN}(x)=\max(0,xW_1+b_1)W_2+b_2

base 模型中 d_model=512，d_ff=2048。后续大模型把这个模块扩展成更大的 MLP、门控 FFN 或 MoE，本质上仍沿用“attention 做路由，FFN 做逐 token 计算”的分工。

5. Positional Encoding：没有 recurrence 后如何知道顺序

Self-attention 本身对位置置换不敏感，所以模型必须显式接收位置信息。论文主实验使用固定正弦/余弦位置编码，把它与 token embedding 相加。

论文 Section 3.5
\[PE_{(pos,2i)}=\sin\left(pos/10000^{2i/d_{model}}\right)\]
\[PE_{(pos,2i+1)}=\cos\left(pos/10000^{2i/d_{model}}\right)\]
LaTeX: PE_{(pos,2i)}=\sin(pos/10000^{2i/d_{model}}), PE_{(pos,2i+1)}=\cos(pos/10000^{2i/d_{model}})

作者选择 sinusoidal encoding 的理由是固定 offset 的位置关系可以由当前位置编码线性表示，可能更容易外推到更长序列。Table 3(E) 显示 learned positional embedding 为 25.7 BLEU，sinusoidal base 为 25.8 BLEU，差距很小；因此本文的主结论不是“某种位置编码魔法”，而是 attention 主干本身。

实验设置

论文主要验证 WMT 2014 English-German 和 English-French 机器翻译。EN-DE 约 450 万句对，共享约 37K BPE 词表；EN-FR 约 3600 万句对，使用 32K word-piece 词表。每个训练 batch 约包含 25K source tokens 和 25K target tokens。

核心指标BLEU 越高越好；Table 3 的 perplexity 越低越好，但不能和 BLEU 完全等价。
训练成本论文报告 8 张 NVIDIA P100；base 约 12 小时，big 约 3.5 天。
读表重点Table 2 看质量与训练成本；Table 3 看架构消融；Table 1 看复杂度与路径长度。
论文 Equation (3)
\[\mathrm{lrate}=d_{model}^{-0.5}\cdot\min\left(\mathrm{step\_num}^{-0.5},\mathrm{step\_num}\cdot\mathrm{warmup\_steps}^{-1.5}\right)\]
LaTeX: \mathrm{lrate}=d_{model}^{-0.5}\cdot\min(\mathrm{step\_num}^{-0.5},\mathrm{step\_num}\cdot\mathrm{warmup\_steps}^{-1.5})

这个 schedule 先线性 warmup，再按 step 的平方根倒数衰减。复现时，学习率 warmup、label smoothing、dropout、checkpoint averaging 和 beam search 都是结果的一部分，不能只照抄网络结构。

消融分析

Q1：多头是否真的必要？

Table 3(A) 在近似固定计算量下改变 head 数：1 head 为 24.9 BLEU，4 heads 为 25.5，8 heads base 为 25.8，16 heads 仍为 25.8，32 heads 降到 25.4。结论是多头能表达多个关系子空间，但每头维度太小也会伤害兼容性计算。

Q2：宽度、深度和 FFN 容量贡献什么？

Table 3(C) 显示减少层数或模型维度会明显降分；增大模型维度和 FFN 隐层可以提升 dev BLEU，但也增加参数和计算。这里的证据支持“attention 主干需要足够的通道容量”，而不是仅靠 attention 算子本身。

Q3：正则化和位置编码是否关键？

Table 3(D) 中 dropout=0.0 时 BLEU 降到 24.6，说明正则化是大模型训练的必要组成部分。Table 3(E) 中 learned positional embedding 为 25.7，sinusoidal base 为 25.8，几乎持平；因此位置编码实现不是本文主结论的唯一来源。

替代解释：这些消融主要在 EN-DE dev set 上完成，且超参之间存在耦合。它们证明设计选择合理，但不能证明这些设置在所有任务、长度分布和模型规模上都是最优。

优点、不足与隐含假设

优点训练并行度高；任意位置路径短；模块边界清楚；矩阵乘法友好；实验同时报告质量、训练成本和架构消融。
不足self-attention 对长度二次复杂度；自回归解码仍串行；证据主要集中在句子级翻译；长文档、音频、图像等只是未来工作。

隐含假设包括：句子级长度 n 通常足够短，使 O(n² · d) 可接受；BLEU 足以代表翻译质量；显式位置编码加内容寻址足以替代 recurrence；训练和部署长度分布足够接近。这些假设在后续长上下文、流式语音、代码、视觉和机器人任务中都会被重新检验。

后续研究与工程启发

研究上，Transformer 把“内容寻址 + 并行矩阵计算”变成通用序列建模范式，后续 BERT、GPT、T5、ViT 等都在这个骨架上扩展。开放问题也很清楚：如何降低长序列 attention 成本，如何改进位置/结构归纳偏置，如何让非自回归或半自回归生成更可靠，如何解释和约束 attention head 的行为。

工程上，复现优先检查 attention mask、学习率 warmup、label smoothing、dropout、checkpoint averaging 和 beam search。部署时重点监控长序列显存、KV cache、生成延迟以及不同长度分布上的质量退化。Transformer 的优势来自高并行训练，但在线生成仍需要逐 token 解码，这一点不应被训练速度掩盖。

总结

《Attention Is All You Need》的核心贡献，是把 attention 从辅助对齐模块升级为完整序列建模主干。最有说服力的证据是：Table 2 显示 Transformer 在 WMT14 翻译上以较低训练成本达到强结果，Table 1 解释了路径长度和并行度优势，Table 3 说明多头、宽度、正则化和位置编码的作用边界。最大局限是二次 attention 成本和自回归推理串行性仍在；后续多年大量工作基本都围绕这两条线继续推进。
