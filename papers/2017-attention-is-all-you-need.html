Attention Is All You Need

这篇论文的核心贡献不是“用了注意力”，而是证明了：在序列转导任务中，编码器和解码器可以完全摆脱循环和卷积，只用多头自注意力、前馈网络、残差归一化与位置编码，就同时获得更强的质量、更短的训练时间和更好的并行性。

一句话贡献Transformer 用全注意力架构替代 RNN/CNN 序列骨干，使远距离依赖路径长度降到常数。
研究问题如何在机器翻译中同时建模长程依赖、保持自回归生成，并避免 RNN 的串行训练瓶颈。
主要结果WMT14 EN-DE 28.4 BLEU；EN-FR 41.8 BLEU；base 模型 8 张 P100 约 12 小时训练。

背景与动机：为什么需要这篇论文？

论文出现之前，机器翻译等 sequence transduction 任务的主流 pipeline 是 encoder-decoder：编码器把输入句子压成一串连续表示，解码器自回归地产生目标句子。强模型通常是 RNN/LSTM/GRU 或 CNN，再叠加 attention 让解码器能看输入不同位置。

关键问题是：RNN 把位置和计算时间绑定在一起。第 \(i\) 个 hidden state 依赖第 \(i-1\) 个 hidden state，因此同一个样本内部很难并行。CNN 可以并行，但两个远位置之间的信息交互需要多层卷积传播，路径长度随距离增长。Transformer 的动机就是把“任意两个 token 建立依赖”变成一层内完成的操作。

理解入口：这篇论文真正解决的是训练和建模路径问题。它不是给 RNN 加一个更强 attention，而是把 attention 提升为主干计算图。

核心瓶颈：序列模型为什么难以并行又难学长依赖？

输入 / 输出输入是 token 序列 \((x_1,...,x_n)\)，输出是目标 token 序列 \((y_1,...,y_m)\)。模型需要在生成第 \(t\) 个输出时使用源句和已生成目标前缀。
失败模式RNN 的顺序依赖限制训练并行；CNN 的感受野需要层数堆叠；普通 dot-product attention 在维度大时 softmax 容易饱和。

作者的建模假设是：序列中任意位置之间的依赖可以通过内容寻址完成，顺序信息不必由 recurrence 承载，而可以作为位置编码显式加入表示。这个假设后来成为大语言模型架构的起点。

对应论文 Figure 1：Transformer 模型架构。左侧 encoder block 重复 6 层，右侧 decoder block 重复 6 层。它支持的核心结论是：序列表示更新可以由 self-attention + FFN 组成，不需要 recurrent state。

完整 pipeline 可以分成四步：第一，输入 token 和右移后的目标 token 被映射为 embedding，并加上 positional encoding；第二，encoder 用多头 self-attention 和逐位置 FFN 反复更新源序列表示；第三，decoder 先用 masked self-attention 只看目标前缀，再用 encoder-decoder attention 读取源句表示；第四，线性层和 softmax 输出下一个 token 概率。

核心方法精读

1. Scaled Dot-Product Attention：把对齐变成矩阵乘法

注意力函数把 query 与一组 key-value 映射到输出。Transformer 把所有 query 打包成矩阵 \(Q\)，key 和 value 打包成 \(K,V\)，从而一次矩阵乘法完成所有位置之间的兼容性计算。

论文 Equation (1)
\[\mathrm{Attention}(Q,K,V)=\mathrm{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V\]

这里 \(QK^T\) 给出每个 query 对每个 key 的打分，\(\sqrt{d_k}\) 是缩放因子。没有缩放时，若 \(d_k\) 较大，点积方差变大，softmax 进入梯度很小的区域；缩放相当于控制 logits 温度，让训练更稳定。工程上，这个公式就是两次矩阵乘法加一个 softmax，因此非常适合 GPU。

对应论文 Figure 2：左图是 scaled dot-product attention，右图是 multi-head attention。图中最重要的是两点：mask 在 softmax 前生效；多头注意力先分头投影、并行注意力、再拼接投影。

2. Multi-Head Attention：用多个低维子空间避免单头平均

单个 attention head 会把多个关系压进一次加权平均。多头注意力把 \(Q,K,V\) 投影到多个子空间，让不同 head 可以分别捕获语法依赖、局部位置、指代关系或对齐模式。

论文 Section 3.2.2
\[\mathrm{MultiHead}(Q,K,V)=\mathrm{Concat}(head_1,\ldots,head_h)W^O,\quad head_i=\mathrm{Attention}(QW_i^Q,KW_i^K,VW_i^V)\]

base 模型使用 \(h=8\)，\(d_{model}=512\)，每个 head 的 \(d_k=d_v=64\)。由于每个 head 降维，总计算量与单头全维 attention 接近。Table 3 的消融显示，1 个 head 比 base 低 0.9 BLEU；但 head 太多且每头维度过小也会下降，说明“多头”有效，但不是越多越好。

3. Encoder / Decoder 的三类注意力

Transformer 里 attention 有三种用途：encoder self-attention 让源句每个位置看所有源位置；decoder masked self-attention 让目标位置只看自己和历史目标；encoder-decoder attention 让 decoder query 读取 encoder 产生的源句 memory。mask 的作用不是正则化，而是保持自回归因果性。

4. Position-wise FFN：每个位置共享的非线性变换

论文 Equation (2)
\[\mathrm{FFN}(x)=\max(0,xW_1+b_1)W_2+b_2\]

attention 负责跨位置混合，FFN 负责对每个位置的通道表示做非线性变换。它等价于 kernel size 为 1 的两层卷积：位置之间不通信，但所有位置共享参数。base 模型中输入输出维度为 512，隐藏维度为 2048。

5. Positional Encoding：没有 recurrence 后，顺序从哪里来？

论文 Section 3.5
\[PE_{(pos,2i)}=\sin\left(pos/10000^{2i/d_{model}}\right),\quad PE_{(pos,2i+1)}=\cos\left(pos/10000^{2i/d_{model}}\right)\]

位置编码与 token embedding 相加，使模型既看到内容，也看到位置。作者选择正弦位置编码，是因为任意固定偏移 \(k\) 的位置编码可由当前位置编码线性表示，这为学习相对位置关系提供便利。Table 3 row (E) 显示 learned positional embedding 与 sinusoidal encoding 的 BLEU 几乎相同，作者最终选择正弦版本，理由是可能外推到更长序列。

实验设置：如何读这些实验？

主要实验是 WMT 2014 EN-DE 和 EN-FR 机器翻译。EN-DE 使用约 450 万句对、共享约 37K BPE 词表；EN-FR 使用约 3600 万句对、32K word-piece 词表。训练 batch 约含 25K 源 token 和 25K 目标 token。指标 BLEU 越高越好，训练成本 FLOPs 越低越好。

论文 Equation (3)：学习率 schedule
\[lrate=d_{model}^{-0.5}\cdot\min(step\_num^{-0.5}, step\_num\cdot warmup\_steps^{-1.5})\]

这个 schedule 先 warmup，再按 step 的平方根倒数衰减。base 模型训练 100K steps 约 12 小时；big 模型训练 300K steps 约 3.5 天。正则化包括 residual dropout、embedding/position dropout 和 label smoothing。

消融分析：每个变化在验证什么？

Q1：多头是否真的必要？

Table 3(A) 固定计算量改变 head 数。1 head 的 BLEU 为 24.9，base 为 25.8；16 heads 仍为 25.8，但 32 heads 下降到 25.4。解释是：多头提供多个表示子空间，但每头维度过小会削弱 compatibility 计算。

Q2：key/value 维度和模型宽度有多重要？

Table 3(B)(C) 显示减小 attention key size 或减少层数会损害性能；扩大模型到更高 \(d_{model}\) 和更大 FFN 维度可提升 BLEU，但参数也明显增加。也就是说，Transformer 的性能来自架构和规模共同作用。

Q3：正则化和位置编码选择是否关键？

Table 3(D) 显示 dropout 对避免过拟合很重要。Table 3(E) 中 learned positional embedding 的 dev BLEU 为 25.7，base 正弦位置编码为 25.8，差异很小。这说明论文主结论不依赖某种特定位置编码技巧。

替代解释：消融主要在 EN-DE dev set 上完成，且很多超参变化互相耦合。它支持设计选择的合理性，但不能单独证明每个模块在所有任务上都是最优。

优点、不足与隐含假设

优点极高并行性；任意位置一层交互；矩阵乘法友好；模块边界清楚；实验同时比较质量和训练成本。
不足self-attention 对长度是二次复杂度；推理仍自回归逐步生成；主要实验集中在机器翻译；长文档、音频、图像等场景仅作为未来工作提出。

隐含假设包括：训练/测试分布相近；BLEU 能代表翻译质量；句子级序列长度足够短，使 \(O(n^2d)\) 可接受；位置编码加内容注意力足以替代显式 recurrence。

对后续研究和工程实践的启发

研究上，Transformer 把“内容寻址 + 并行计算”变成通用序列建模范式，直接启发了 BERT、GPT、T5 等后续模型。工程上，复现时最关键的是 attention mask、学习率 warmup、残差 dropout、label smoothing 和 checkpoint averaging；这些训练细节和架构本身一样影响最终指标。

部署时需要额外关注两个问题：长序列显存和延迟，因为 attention 矩阵随长度平方增长；自回归解码吞吐，因为训练可并行不等于生成可完全并行。

总结

《Attention Is All You Need》最重要的贡献，是把 attention 从辅助对齐模块升级为完整的序列建模骨干。最有说服力的证据来自 Table 2：在 WMT14 翻译任务上，Transformer 同时获得更高 BLEU 和更低训练成本。最大局限是二次复杂度和自回归解码仍然存在。后续研究的主线基本沿着这两个方向展开：更高效的 attention，以及更强的生成训练与推理机制。
